## Key Concepts

### Translation Management

NAB AL Tools provides XLIFF translation management for Business Central:

- **g.xlf file** - Generated by AL compiler, contains all translatable texts
- **Language XLF files** - One per target language with translations
- **Translation modes** - Choose between NAB Tags (simple) or Target States (professional)
- **Translation matching** - Reuse previous translations automatically
- **BaseApp integration** - Leverage Microsoft's standard translations

### Translation Modes

**NAB Tags Mode (Default):**

- Uses visible text markers: `[NAB: NOT TRANSLATED]`, `[NAB: REVIEW]`, `[NAB: SUGGESTION]`
- Simple and visual
- Best for: In-house translation, VS Code-only workflow

**Target States Mode:**

- Uses XLIFF standard state attributes
- Professional XLIFF workflow
- Best for: External tools, professional translators, AI workflows

**Learn more:** [Target States vs NAB Tags](features/target-states-vs-nab-tags.md)

### AI-Assisted Translation

**NAB-XLF-Translator Agent:**

- Automated translation workflow using GitHub Copilot
- Applies glossaries automatically
- Quality validation built-in
- See the main [extension README](../extension/README.md#nab-xlf-translator-agent) for details

## Common Workflows

### Basic Translation Workflow

1. Write AL code with labels and captions
2. Build application to generate g.xlf
3. Run **`NAB: Refresh XLF files from g.xlf`**
4. Press `Ctrl+Alt+U` to find untranslated texts
5. Translate each text
6. Repeat until complete

**See:** [Translation Workflow Guide](guides/translation-workflow.md)

### AI Translation Workflow

1. Open file from your AL app
2. Run `/translateXlfFiles` in GitHub Copilot Chat
3. Agent handles everything automatically
4. Review completed translations

**See:** [Translation Workflow Guide](guides/translation-workflow.md)

### Review Workflow

1. Run `/reviewXlfFiles` in GitHub Copilot Chat
2. Agent reviews translations for quality
3. Suggests improvements
4. Updates states appropriately

**See:** [Translation Review Workflow](guides/translation-review-workflow.md)

## Essential Commands

### Translation

- **`NAB: Refresh XLF files from g.xlf`** - Update all language files from g.xlf
- **`NAB: Find next untranslated text`** (`Ctrl+Alt+U`) - Navigate to next item needing attention
- **`NAB: Create translation XLF for new language`** - Add support for new language

### Experimental Commands

These commands are experimental and have limitations:

- **`NAB: Update g.xlf`** ‚ö†Ô∏è _Experimental_ - Update g.xlf from AL code (may not capture all changes)
- **`NAB: Update all XLF files`** ‚ö†Ô∏è _Experimental_ - Update g.xlf and refresh language files (uses experimental g.xlf update)

### Navigation

- **`NAB: Find translated texts of current line`** - View translations of current AL code
- **`NAB: Find source of current Translation Unit`** (`F12` in XLF) - Jump to AL source

### Import/Export

- **`NAB: Export Translations to .csv`** - Export for external editing
- **`NAB: Import Translations from .csv`** - Import translated CSV
- **`NAB: Import Translations by Id`** - Import from another XLF file

### Matching

- **`NAB: Match Translations From Base Application`** - Use Microsoft's BC translations
- **`NAB: Match translations from external XLF file`** - Reuse external translations

## Key Settings

> **Team Configuration:** For team projects, configure NAB translation settings in your `.code-workspace` file to ensure all developers use the same translation mode and workflow. This prevents conflicts between NAB tags and target states modes.

### Translation Mode

```json
{
  // Enable XLIFF state attributes (false = NAB tags mode)
  // Default: false
  "NAB.UseTargetStates": false
}
```

### Translation Matching

```json
{
  // Enable automatic matching of previous translations
  // Default: true
  "NAB.MatchTranslation": true,

  // Auto-accept first match (NAB tags mode only)
  // Default: false
  "NAB.AutoAcceptSuggestions": false,

  // State for exact matches (target states mode)
  // Default: "" (no state change)
  "NAB.SetExactMatchToState": ""
}
```

### User Experience

```json
{
  // Show translations on hover in AL code
  // Default: true
  "NAB.EnableTranslationsOnHover": true,

  // Highlight translation issues
  // Default: true
  "NAB.ShowXlfHighlights": true
}
```

**See:** [Settings Reference](reference/settings.md) for all settings

## Best Practices

### Translation Quality

1. **Use glossaries** - Ensure consistent terminology
2. **Translate in context** - Understand how text is used
3. **Preserve placeholders** - Keep %1, %2, %3 intact
4. **Test in BC** - Verify translations in running application
5. **Version control** - Commit XLF files with code changes

### Workflow Efficiency

1. **Use keyboard shortcuts** - `Ctrl+Alt+U` is your friend
2. **Enable matching** - Reuse previous translations
3. **Leverage BaseApp** - Use Microsoft's standard translations
4. **Create glossaries** - Document project-specific terms
5. **Try AI assistance** - Automate repetitive translation work

### Team Collaboration

1. **Choose one mode** - Standardize on NAB tags or target states
2. **Document decisions** - Record translation conventions
3. **Share glossaries** - Maintain in version control
4. **Review process** - Establish quality standards
5. **Regular updates** - Keep translations current with code
